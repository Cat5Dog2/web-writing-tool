# 画面設計書

## 1. 目的

本書は、AIライティングツールのBlazor Web App画面仕様を定義する。対象は、ログイン、共通レイアウト、記事一覧、記事作成、生成結果編集、設定、WordPress投稿、一括登録、管理者画面である。

画面は業務アプリとして、情報密度、操作の見通し、生成ジョブの状態把握、記事編集の効率を優先する。

## 2. 画面設計方針

- Blazor Web Appを採用する。
- 一覧、設定、記事編集など操作が多い画面はInteractive Serverを使用する。
- 画面コンポーネントから直接DBへアクセスしない。ApplicationサービスまたはAPIを呼び出す。
- 長時間処理は画面で待たず、ジョブ登録後にステータスを表示する。
- UI上の表示制御に加えて、API/Applicationサービス側で認可を必ず検証する。
- 管理画面型の落ち着いたUIとし、フォーム、テーブル、タブ、モーダルを中心に構成する。
- 画面幅が狭い場合は2カラムを1カラムに折り返す。

## 4. 共通レイアウト

### 4.1 構成

| コンポーネント | 役割 |
| --- | --- |
| `MainLayout` | 共通レイアウト |
| `AppHeader` | ロゴ、ユーザーメニュー |
| `TopNavigation` | 主要機能ナビゲーション |
| `UsageSummary` | 文字数上限設定、利用モデル、残り構成数 |
| `PageTabs` | 記事作成内のプロジェクト/詳細/設定タブ |
| `AppFooter` | コピーライト |
| `ToastHost` | 成功/失敗通知 |
| `ModalHost` | モーダル表示 |

### 4.2 ヘッダー

表示項目:

- ロゴ
- ユーザーアイコン
- ユーザーメニュー
  - アカウント情報
  - 退会
  - ログアウト

操作:

- ロゴ押下で記事一覧へ遷移する。
- ユーザーアイコン押下でメニューを開閉する。
- 退会押下で退会確認モーダルを表示する。
- ログアウト押下でセッションを終了し、ログイン画面へ遷移する。

退会確認モーダル:

- 対象ユーザーに紐づく記事、ジョブ、検索結果、X投稿キャッシュ、WordPress連携、通知設定、利用履歴、対象ユーザーが操作した既存監査ログが物理削除されることを明示する。
- 現在パスワードと確認文字列`DELETE`の入力を必須にする。
- `Running`ジョブがある場合は退会ボタンを無効化し、ジョブ完了後に再実行する案内を表示する。
- 最後のAdminユーザーの場合は退会ボタンを無効化する。
- 退会成功後はログイン画面へ遷移する。

### 4.3 主要ナビゲーション

表示項目:

- キーワード発掘
- リサーチ
- 記事作成
- 画像生成
- リライト

MVPでは主要ナビゲーション上のキーワード発掘、リサーチ、画像生成、リライトは無効状態または後続フェーズ案内とする。画像生成はMVPに含めない。リライト処理自体は、生成結果編集画面内の本文編集操作として提供する。

### 4.4 利用上限表示

表示項目:

| 項目 | 表示例 |
| --- | --- |
| 文字数上限設定 | `上限 200,000文字` |
| 対象モデル | `(Gemini 3.1 Pro Preview)` |
| 残り構成数 | `/40構成` |

取得元:

- `GET /api/usage/summary`

MVPでは月次利用量集計と残量算出は扱わない。文字数上限は設定値として表示するだけで、利用文字数消費による残量表示には使わない。

表示タイミング:

- ページ初期表示時
- ジョブ完了後
- AI生成操作後

## 8. `SCR-020` 記事作成

### 8.1 目的

単体記事のキーワード、タイトル、詳細条件を入力し、見出し構成生成ジョブを作成する。

### 8.2 レイアウト

上から以下を配置する。

1. ページタイトル `記事作成`
2. タブ: `プロジェクト` / `詳細` / `設定`
3. 基本入力行
4. 詳細設定パネル
5. 下部設定行
6. 構成作成ボタン
7. ジョブ状態表示

### 8.3 基本入力

| 項目 | 種別 | 必須 | 制約 | 備考 |
| --- | --- | --- | --- | --- |
| キーワード | text | 必須 | 1から200文字 | |
| 記事タイトル候補を出す | button | 任意 | | タイトル候補生成 |
| タイトル | text | 任意 | 250文字以内 | |

### 8.4 詳細設定

詳細設定は初期表示で折りたたみ可能とする。

| 項目 | 種別 | 必須 | 制約 |
| --- | --- | --- | --- |
| H2の個数 | number | 任意 | 1から20 |
| H3の個数 | number | 任意 | 0から60 |
| 文章のトーン | select | 任意 | `通常`など |
| 情報は日本国内のものに限定 | checkbox | 必須 | 既定true |
| タグ | text | 任意 | カンマ区切り |
| メモ | text | 任意 | 1000文字以内 |
| サジェストキーワード | textarea | 任意 | 10000文字以内 |
| 関連キーワード | textarea | 任意 | 10000文字以内 |
| 事前学習種別 | radio | 任意 | テキスト / URL |
| 事前学習本文 | textarea | 任意 | 設定上限内 |
| 追加プロンプト | textarea | 任意 | 3000文字以内 |
| サイト別ライティング設定 | select | 任意 | WordPressサイトを選択 |

### 8.5 下部設定

| 項目 | 種別 | 必須 | 備考 |
| --- | --- | --- | --- |
| 見出し構築方法 | select | 必須 | キーワード/検索/AI |
| 何を作成する？ | select | 必須 | モデル選択 |
| 検索モード | checkbox | 必須 | Web検索利用 |
| 通知設定 | select | 任意 | 通知なし/Discord |

### 8.6 操作

| 操作 | API | 成功時 |
| --- | --- | --- |
| タイトル候補を出す | `POST /api/articles/{articleId}/generation/title-candidates` | タイトル候補選択モーダル表示 |
| 構成を作成 | `POST /api/articles/{articleId}/generation/outline` | ジョブ状態表示、編集画面へ誘導 |
| 保存 | `PUT /api/articles/{articleId}` | トースト表示 |

新規記事でタイトル候補や構成作成を行う場合、先に`POST /api/articles`で下書き記事を作成する。

### 8.7 バリデーション

- キーワード未入力時は構成作成不可。
- 事前学習種別がURLの場合、HTTPS URLのみ許可する。
- 残り構成数などMVPで制御する残数が不足している場合は構成作成ボタンを無効にし、理由を表示する。月次利用文字数による無効化は行わない。

## 10. `SCR-030` 生成結果編集

### 10.1 目的

生成済み見出しと本文を編集し、HTML変換、WordPress投稿、保存を行う。

### 10.2 レイアウト

デスクトップでは左右2カラムとする。

左カラム:

- 記事作成時のAIモデル選択
- 検索モード
- メタディスクリプション
- 記事全体操作ボタン
- 見出し構成ツリー

右カラム:

- 選択見出しタイトル
- 本文編集エリア
- 本文操作ボタン
- 文字数表示
- ジョブ状態

モバイルでは右カラムを下に配置する。

### 10.3 アイキャッチ画像領域

MVPではこの領域を表示しない。アイキャッチ画像の作成、外部画像URLの保存・表示、画像加工、画像メタデータ保存は後続フェーズで扱う。

| 項目 | 種別 | 説明 |
| --- | --- | --- |
| 画像URL | text | 後続フェーズ |
| 変更 | button | 後続フェーズ |
| 画像に文字を入れる | button | 後続フェーズ |
| 画像プレビュー | image | 後続フェーズ |

### 10.4 記事全体操作

| ボタン | 処理 |
| --- | --- |
| 一括取得 | 全見出し本文生成 |
| 一括取得(H3以下) | H3のみ本文生成 |
| HTML変換 | 記事本文をHTMLへ変換 |
| WordPress投稿 | WordPress投稿モーダル |
| 保存 | 記事と見出しを保存 |

### 10.5 見出し構成ツリー

表示項目:

| 項目 | 内容 |
| --- | --- |
| 階層 | H2 / H3 |
| 追加 | 同階層または子見出し追加 |
| 削除 | 見出し削除 |
| タイトル | 見出しタイトル |
| 文字数 | 実文字数と目安 |
| 上下移動 | 並び替え |
| 設定 | 見出し設定 |
| Web検索 | 見出し単位の検索適用 |

操作:

- 見出し行クリックで右カラムに本文を表示する。
- H2削除時は配下H3も削除対象とし、確認を表示する。
- 生成中の見出しは削除、並び替えを無効にする。

### 10.6 本文編集領域

| 項目 | 種別 | 説明 |
| --- | --- | --- |
| 見出しタイトル | text | 選択中見出し |
| 本文 | textarea | 選択中見出し本文 |
| 文字数 | label | 現在文字数 |
| ステータス | badge | Pending/Queued/Generating/Generated/Failed |

操作:

| ボタン | 処理 |
| --- | --- |
| 再取得 | 本文を再生成 |
| 下階層を要約 | 配下H3を要約 |
| 要約 | 選択本文を要約 |
| 長文化 | 選択本文を長文化 |
| リライト | 選択本文をリライト |
| `。`で改行を挿入 | 本文を整形 |

### 10.7 保存・未保存制御

- 本文、タイトル、見出し順、メタディスクリプション変更時に未保存状態にする。
- 未保存のまま別見出しへ移動する場合は保存、破棄、キャンセルを選択させる。
- 未保存のまま画面遷移する場合は確認モーダルを表示する。
- MVPでは本文履歴、差分表示、過去版復元UIを提供しない。保存、再取得、要約、長文化、リライト、HTML変換は現在本文を更新する。

### 10.8 ジョブ状態表示

| 状態 | 表示 |
| --- | --- |
| Queued | `生成待ち` |
| Running | スピナーと`生成中` |
| Succeeded | `完了` |
| Failed | エラー概要と再実行ボタン |
| Canceled | `キャンセル済み` |

ジョブ状態は数秒間隔でポーリングする。将来的にSignalRへ置き換え可能にする。

## 11. `MDL-002` WordPress投稿

### 11.1 目的

生成済み記事をWordPressへ投稿する。

### 11.2 項目

| 項目 | 種別 | 必須 | 制約 |
| --- | --- | --- | --- |
| 投稿先サイト | select | 必須 | 登録済みサイト |
| タイトル | text | 必須 | 250文字以内 |
| カテゴリ | select | 任意 | WordPressカテゴリ |
| 投稿ステータス | select | 任意 | 下書き/公開。初期値は下書き |
| 本文HTML | textarea | 必須 | HTML |

### 11.3 操作

| 操作 | 処理 |
| --- | --- |
| Markdown変換 | MVPでは非表示 |
| note投稿はこちら | MVPでは非表示 |
| 投稿 | `POST /api/articles/{articleId}/wordpress-posts` |
| 人間確認を完了 | `POST /api/articles/{articleId}/human-review` |
| 閉じる | モーダルを閉じる |

### 11.4 表示制御

- 投稿可能ステータス以外の記事では投稿ボタンを無効にする。
- WordPressサイト未登録時は設定画面への導線を表示する。
- 投稿ジョブ登録後はジョブ状態を表示する。
- `HumanReviewRequired = true`かつ`HumanReviewedAt`が未設定の場合、公開投稿は無効にし、下書き投稿のみ許可する。
- 人間確認が必要な記事では、公開投稿の近くに確認完了操作を表示する。確認完了後に本文、HTML本文、タイトル、メタディスクリプション、X引用を変更した場合は未確認状態に戻す。

## 14. `SCR-060` ユーザー管理

### 14.1 目的

管理者がユーザー作成、ロール、有効状態、利用上限、ユーザー削除を管理する。

### 14.2 項目

| 列 | 内容 |
| --- | --- |
| メールアドレス | ユーザーID |
| 表示名 | DisplayName |
| ロール | Admin/User |
| 有効状態 | 有効/無効 |
| 月間上限 | 文字数 |
| 操作 | 編集、ロール変更、無効化、削除 |

MVPでは月間上限を設定値として表示する。利用済み文字数、残り文字数、課金金額、請求情報は表示しない。

### 14.3 ユーザー追加

- 管理者はユーザーを新規作成できる。
- 入力項目はメールアドレス、表示名、初期パスワード、ロール、利用上限、有効状態とする。
- ロールは`User`を既定値とし、必要な場合のみ`Admin`を選択する。
- 初期パスワードは登録後に再表示しない。
- 作成成功後はユーザー一覧を再読み込みする。

### 14.4 ロール変更

- 管理者は既存ユーザーをAdminへ昇格できる。
- 管理者はAdminをUserへ降格できる。
- 最後のAdminユーザーは降格できない。
- ロール変更時は確認モーダルを表示する。

### 14.5 表示名・有効状態・利用上限変更

- 管理者は表示名、有効状態、月間文字数上限、残り構成生成回数を更新できる。
- MVPではメールアドレス変更とパスワード再設定は扱わない。
- 最後のAdminユーザーは無効化できない。
- 残り構成生成回数を0にした場合、新規構成生成ジョブを登録不可として扱う。
- 月間文字数上限はMVPでは設定値表示用であり、文字数消費に基づくジョブ拒否には使わない。

### 14.6 ユーザー削除

- 削除操作はAdminロールのみ表示する。
- 管理者自身と最後のAdminユーザーには削除ボタンを表示しない。
- 削除確認モーダルでは、対象ユーザーに紐づく記事、ジョブ、検索結果、X投稿キャッシュ、WordPress連携、通知設定、利用履歴、対象ユーザーが操作した既存監査ログが物理削除されることを明示する。
- 対象ユーザーに`Running`ジョブがある場合、削除ボタンを無効化し、ジョブ完了後に再実行する案内を表示する。
- 削除成功後はユーザー一覧を再読み込みする。

### 14.7 監査ログ

- ユーザー作成、表示名/有効状態変更、ロール変更、利用上限変更、ユーザー削除を一覧で確認できる。
- 削除済みユーザーはFKではなく対象ユーザーIDの文字列スナップショットとして表示する。
- パスワード、秘密情報、記事本文全文は表示しない。

### 14.8 MVP対象外

- 失敗ジョブを全ユーザー横断で確認・復旧する管理者画面はMVPでは提供しない。
- 失敗ジョブの確認と再実行は、記事詳細・生成結果画面のジョブ状態表示から行う。
- 管理者向けの失敗ジョブ一覧、強制再実行、Running期限切れ復旧、ジョブ詳細Payloadの安全な確認は後続フェーズで検討する。

- エラーメッセージはスクリーンリーダーで認識できるようにする。

## 21. MVP画面範囲と後続フェーズ

### 21.1 キーワード発掘・リサーチ・リライト

MVPでは記事作成を主導線とし、キーワード発掘、リサーチ、リライトの独立画面は提供しない。記事作成ワークフロー内で必要な最小機能だけを扱う。

| 項目 | MVP方針 | 後続フェーズ |
| --- | --- | --- |
| キーワード発掘 | 独立画面は提供しない。記事作成画面のキーワード入力、一括登録、タイトル候補生成で対応する。 | 検索ボリューム、関連語、競合分析、候補保存を持つ専用画面を追加する。 |
| リサーチ | 独立画面は提供しない。記事作成詳細の検索モード、事前学習テキスト/URL、見出し単位Web検索に内包する。 | Tavily/X検索結果の比較、引用候補管理、調査メモ、再取得操作を持つ専用画面を追加する。 |
| リライト | 独立画面は提供しない。生成結果編集画面で選択本文に対する要約、長文化、リライトを提供する。 | 複数記事横断のリライト、差分比較、版管理、トーン一括変換を持つ専用画面を追加する。 |

### 21.2 画像生成・note投稿・ライター管理

画像生成、note投稿、ライター管理はMVPでは扱わず、後続フェーズで以下の順に追加する。初回追加時は既存の記事作成ワークフローに近い範囲へ絞り、専用画面や高度な管理機能は段階的に追加する。

| 項目 | 追加順 | 初回追加時の画面範囲 | 後続拡張 |
| --- | --- | --- | --- |
| 画像生成 | Phase 2 | 生成結果編集画面にアイキャッチ画像領域を追加する。画像生成、再生成、プレビュー、選択、WordPress投稿時のアイキャッチ指定を扱う。 | 独立した画像生成画面、画像一覧、画像に文字を入れる機能、テンプレート、トリミングを追加する。 |
| note投稿 | Phase 2.5 | WordPress投稿モーダルとは別にnote投稿用変換モーダルを追加する。タイトル、本文、タグ、Markdown/HTML変換、コピー操作を扱う。 | note直接投稿、投稿履歴、再投稿、note向け装飾調整を追加する。 |
| ライター管理 | Phase 3 | 管理者画面にライター管理を追加し、記事割当、担当記事一覧、納品ステータスを扱う。 | コメント、レビュー依頼、差し戻し、権限細分化、ライター別実績表示を追加する。 |

### 21.3 画像に文字を入れる機能

後続フェーズの初回実装では、自由編集型の画像エディタではなく、アイキャッチ画像へ記事タイトルを重ねるためのプリセット型UIとして提供する。

| 項目 | 方針 |
| --- | --- |
| 起動元 | 生成結果編集画面のアイキャッチ画像領域 |
| 表示形式 | モーダルまたは右サイドパネル |
| 対象画像 | 選択済みアイキャッチ画像 |
| テキスト | 記事タイトルを初期値とし、任意編集できる |
| レイアウト | 中央、下部、左下、帯付き下部などのプリセットから選択 |
| 装飾 | フォントサイズ、太字、文字色、背景帯色、背景帯透明度 |
| プレビュー | 16:9固定プレビュー |
| 保存 | 元画像を保持し、文字入れ済み画像を別アセットとして保存 |
| 対象外 | 自由ドラッグ、複数テキストレイヤー、Undo/Redo、複雑なトリミング、フィルター |

### 21.4 記事インポートのファイル形式

後続フェーズの初回実装では、記事インポートは完成記事の移行機能ではなく、一括登録のファイル版として扱う。

| 項目 | 方針 |
| --- | --- |
| 対応形式 | CSV |
| 文字コード | UTF-8。UTF-8 BOM付きも許可 |
| ヘッダー | 必須 |
| 必須列 | `keyword` |
| 任意列 | `title`, `tags`, `memo` |
| 一括設定 | H2/H3数、検索モード、通知設定、WordPress自動投稿設定は画面側で指定 |

### 21.5 生成結果編集の保存方針

初期実装ではサーバー側の自動保存は行わず、明示的な保存操作を基本とする。未保存変更の喪失とAI生成結果による意図しない上書きを避けるため、見出し切替、画面遷移、ジョブ操作前に確認を挟む。

| 項目 | 方針 |
| --- | --- |
| 保存方式 | 保存ボタンによる明示保存 |
| 未保存検知 | 本文、見出しタイトル、見出し順、メタディスクリプション変更時に未保存状態にする |
| 見出し切替 | 未保存変更がある場合は保存、破棄、キャンセルを選択させる |
| 画面遷移 | 未保存変更がある場合は確認モーダルを表示する |
| ジョブ操作前 | 再取得、要約、長文化、リライト、HTML変換の前に未保存変更を確認する |
| 競合制御 | 保存時に`RowVersion`で同時更新を検出する |
| 対象外 | 初期実装ではサーバー自動保存、ローカル下書き復元、版管理、差分復元は提供しない |

### 21.6 モバイルでの記事編集方針

MVPではモバイルでも生成結果編集画面を利用可能にするが、主対象はPCまたはタブレットでの編集とする。モバイルでは確認、軽微な本文修正、保存、ジョブ状態確認を中心に扱う。

| 項目 | 方針 |
| --- | --- |
| 主対象 | PCまたはタブレットでの本格編集 |
| モバイル対象 | 記事内容の確認、見出し選択、軽微な本文修正、保存、ジョブ状態確認 |
| レイアウト | 左右2カラムを縦積みにし、見出し構成の下に本文編集領域を表示する |
| 操作ボタン | 折り返し表示し、保存など主要操作を見失わないようにする |
| 本文編集 | textareaを画面幅いっぱいに表示し、最低限の高さを確保する |
| 未保存制御 | PCと同じく、見出し切替・画面遷移前に確認する |
| 対象外 | ドラッグによる快適な見出し並び替え、長文の本格編集、大量一括操作の最適化 |
| 後続対応 | モバイル専用の見出しドロワー、固定保存バー、編集/構成の表示切替を検討する |
